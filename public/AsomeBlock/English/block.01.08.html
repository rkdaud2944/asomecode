  • 2. Preview

    • A. DARTS

    It was a game played by British soldiers in the Middle Ages, throwing a broken bow on the lid of an empty can. Now, it has developed into a world-class leisure sport.


    B. Points of darts

    Darts calculate the score after throwing three darts at a time on a wide dart board.

    Scores are calculated in the following ways.

    ① Single : The number on each fan-shape is the points in its area.

    ② Double : The fan-shaped outer line is twice the basic point.

    ③ Triple : The fan-shaped midline is three times the basic point.

    ④ Bull : The circle in the center of the dartboard is called Bull and it has 50 points.



  • 3. Learn which blocks to use

    • A. Move forward (WALK Group)

    Walk forward by the specified number of steps as an option.

    for i in range(1): asomebot.forward()

    Input the number of steps to move forward after clicking. One step moves left and right foot at a time.


    • Check how many steps the robot need to walk to the center of the dartboard


    B. Move backward (WALK Group)

    Walk backward by the specified number of steps as an option.

    for i in range(1): asomebot.backward()

    Input the number of steps to move backward after clicking. One step moves left and right foot at a time.


    C. Turn Left (WALK Group)

    Turn left as many as specified as an option.

    for i in range(1): asomebot.turn_left()

    Input the number of steps to rotate to the left after clicking. To rotate 90 degrees, the robot should rotate 3 to 4 times.


    • Create a program that rotates 180 degrees in place.


    D. Turn Right (WALK Group)

    Turn right as many as specified as an option.

    for i in range(1): asomebot.turn_right()

    Input the number of steps to rotate to the right after clicking. To rotate 90 degrees, the robot should rotate 3 to 4 times.


    E. Delay (ADVANCE Group)

    Wait for the specified number of seconds to function.

    delay(1)

    Input the number of seconds to wait after a click.


    F. Measure distance (SENSOR Group)

    With an ultrasonic sensor, measure the distance of the obstacle in front. The value of the measured distance is stored in a variable name ‘cm’.

    cm = hcsr04.get_distance()

    G. Repeat infinity (ADVANCE Group)

    [Repeat infinitely] makes blocks that are bundled into it to continuously(endlessly) run in order.

    while True: pass

    • Create a program that the AsomeBot continues to walk forward.
    Press the Stop button at the top when stops walking.


    H. if (ADVANCE Group)

    Execute the block in the block when the detail input as the condition is correct. If the condition is incorrect, the block following the ‘if’ block will be executed.

    ① Input a variable to be used in the conditional statement. Adding sensor blocks to the default value ‘i’ will add the variable and can be selected.

    ② Select the inequality to compare the conditions.

    ③ Input the default value to use in the condition. You need to add blocks to specify the conditional value.

    ④ Add blocks to be executed when the condition is correct.


    ① Create a program that AsomeBot walks 10 steps forward, rotates in place and looks back.

    ② Create a program that after moving forward 8 steps, AsomeBot measures the distance with the ultrasonic sensor and if the distance from the other is equal to or less than 5cm, it moves forward 5 steps.



  • 4. Play activity

    • A. Game rules

    ① Three people (three robots) as a team.

    ② Play Rock, paper, scissors to decide which team to start first.

    ③ The first robot walks around using a program created.

    ④ When the first robot stops, the opposite team's first robot walks around.

    ⑤ After both teams' robots played in order, when all three robots stopped, the team with more points wins.

    B. Algorithm

    ① When you play a dart, there are two critical tactics. The first is to position your robot in a better position. Make your decision up to where your robot should walk referring to the scoreboard. The second tactic is to defend obstacles in order to prevent the counterpart robot from being placed in a good position.

    ② Make some motions that can protect your robot such as, pushing and attacking the counterparts when there is a robot at the location where your robot wants to be or when other robots attack your robot.
